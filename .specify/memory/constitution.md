# TCP/IP Learning Kit Constitution

## コア原則

### I. 学習優先
すべてのコードは、パフォーマンス最適化よりも可読性と教育的価値を優先しなければならない（MUST）。
- 変数名は説明的で自己文書化されていなければならない（MUST）
- 複雑な操作は、理解しやすい小さなステップに分割しなければならない（MUST）
- コメントは「何を」ではなく「なぜ」を説明しなければならない（MUST）

### II. 段階的複雑性
学習教材は、シンプルなものから複雑なものへと構造化された進行に従わなければならない（MUST）。
- 各例は、前の例で紹介された概念の上に構築しなければならない（MUST）
- 新しい概念は一度に1つずつ導入しなければならない（MUST）
- 各レッスンの前提条件は明確に記述しなければならない（MUST）

### III. 実践体験
すべての概念には、実行可能でインタラクティブな例がなければならない（MUST）。
- 例は最小限のセットアップですぐに動作しなければならない（MUST）
- ユーザーはコードを修正して実験できなければならない（MUST）
- 即座のフィードバック（可視化された出力、ログ、またはレスポンス）を提供しなければならない（MUST）

### IV. 明確なドキュメント
すべてのコードと例には包括的なドキュメントを含めなければならない（MUST）。
- READMEファイルは各例が何を実演するかを説明しなければならない（MUST）
- 検証のために期待される出力を文書化しなければならない（MUST）
- トラブルシューティングガイドは一般的な問題に対処しなければならない（MUST）

### V. シンプルさ
実装は概念を実演するために必要な最小限の複雑さを使用しなければならない（MUST）。
- 外部依存関係よりも標準ライブラリを優先しなければならない（MUST）
- 各ファイルは単一の概念に焦点を当てるべきである（SHOULD）
- ボイラープレートコードは最小限にしなければならない（MUST）

## 教育基準

- 対象者: 基本的なプログラミング知識はあるが、ネットワーク経験のない初心者
- 言語: C++（低レベルのソケット操作を直接理解でき、TCP/IPの本質を学べる）
- 環境: クロスプラットフォーム（Linux, macOS, Windows）
- 各レッスンに含める内容: 目的、コード、説明、演習

## ガバナンス

このconstitutionは、TCP/IP Learning Kitのすべての開発決定を統括する。
- 修正には根拠と影響評価の文書化が必要
- すべての貢献は学習優先の原則に沿わなければならない
- コードレビューは正確性だけでなく、教育的明確さを検証しなければならない
